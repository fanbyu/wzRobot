## 🎯 关键发现

通过对比官方库的 `main.ts` 文件，发现了**PWM频率设置**的关键差异！

### 1. PWM频率的重要性

电机驱动板使用PCA9685芯片，不同的设备需要不同的PWM频率：

| 设备类型 | 推荐频率 | 说明 |
|---------|---------|------|
| DC电机 | 150 Hz | 官方库在初始化DC电机时设置 |
| 编码器电机 | 50 Hz | 官方库在控制编码器电机时设置 |
| 舵机 | 50 Hz | 标准舵机PWM频率 |
| IO引脚 | 1000 Hz | 通用IO操作 |

### 修改后的代码

#### 1. DC电机初始化
```typescript

    let motorValue = parameter.MOTOR.code;
    Generator.addObject(`DCmotor_${motorValue}`, 'Emakefun_DCMotor*', 
        `DCmotor_${motorValue} = mMotorDriver.getMotor(${motorValue})`, true);
    Generator.addSetup('mMotorDriver.begin', 'mMotorDriver.begin(150);', true);  // ✅ 添加
}
```

#### 2. 编码器电机初始化
```typescript
export function initEncoderMotor(parameter: any) {
    let encoderValue = parameter.ENCODER.code;
    Generator.addObject(`EncodeMotor_${encoderValue}`, 'Emakefun_EncoderMotor*',

    Generator.addSetup('mMotorDriver.begin', 'mMotorDriver.begin(50);', true);  // ✅ 添加
```

#### 3. 舵机初始化

### 1. 函数签名

**官方库：**
```typescript

```

**我们的库：**


**说明：** `block: any` 参数在Mind+ v1.8.1中不是必需的，去掉后reporter/boolean积木才能正常生成代码。

### 2. 枚举值处理

**官方库：**
```typescript

```

**我们的库：**
```typescript
let motorValue = parameter.MOTOR.code;  // 直接使用 "1"
```

**说明：** 两种方法都可以，因为我们的菜单定义中MOTORS的值直接就是"1", "2", "3", "4"。

### 3. 对象命名

**官方库：**
- DC电机：`DCmotor_1`
- 舵机：`servo1`（小写s，无下划线）
- 编码器：`EncodeMotor_1`

**我们的库：**
- DC电机：`DCmotor_1` ✅
- 舵机：`Servo_1`（大写S，有下划线）
- 编码器：`EncodeMotor_1` ✅

**说明：** 命名风格不同但不影响功能，只要保持一致即可。

### 4. 对象声明方式

**官方库：**
```typescript
Generator.addObject(`mMotorDriver.DCmotor_1`, 'Emakefun_DCMotor', 
    `*DCmotor_1 = mMotorDriver.getMotor(1);`, true);
```

**我们的库：**

**说明：** 我们的方式更清晰，直接声明为指针类型，避免了解引用操作符的混淆。

## 🔧 技术原理

### PCA9685 PWM频率

电机驱动板使用的PCA9685芯片可以产生PWM信号，频率范围通常为40-1600Hz。

**不同设备的最佳频率：**

1. **舵机（Servo）**
   - 标准频率：50Hz（周期20ms）
   - 脉宽范围：0.5-2.5ms
   - 频率过高会导致舵机抖动或不动作

2. **DC电机**
   - 推荐频率：100-200Hz
   - 频率太低会有明显噪音
   - 频率太高会降低扭矩

3. **步进电机**
   - 推荐频率：100-500Hz
   - 取决于电机特性和驱动模式

4. **LED/IO**
   - 可以使用较高频率：1000-1600Hz
   - 减少闪烁

### begin()函数的作用

```cpp
void Emakefun_MotorDriver::begin(uint16_t freq = 1600, TwoWire *theWire = NULL) {
    _freq = freq;
    _pwm.setPWMFreq(freq);  // 设置PCA9685的PWM频率
    // ... 其他初始化
}
```

每次调用`begin()`都会重新设置PWM频率，所以：
- 先初始化DC电机时设置为150Hz
- 再初始化舵机时设置为50Hz
- **最后的设置会覆盖之前的设置**

这就是为什么需要在每个初始化函数中都调用`begin()`的原因！

## 🎓 经验总结

### 1. PWM频率是关键

电机不工作的最常见原因之一就是PWM频率设置不正确。

**排查步骤：**
1. 确认设备类型（DC电机/舵机/步进电机）
2. 查找该设备的推荐PWM频率
3. 在初始化时调用`begin(频率)`设置正确的频率

### 2. 参考官方实现

当遇到问题时，官方库是最好的参考资料：
- 查看官方如何处理相同的功能
- 对比代码生成的差异
- 理解背后的技术原理

### 3. 频率设置的优先级

如果有多种设备共用同一个驱动板：
- **最后调用的begin()会生效**
- 需要根据主要使用的设备类型设置频率
- 或者在使用不同设备前重新设置频率

### 4. Mind+代码生成的特点

- `addSetup()` 会在setup()函数中添加代码
- 同名的setup会被后面的覆盖
- 所以要确保关键的初始化代码在最后执行
