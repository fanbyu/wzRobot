### 2. DC电机对象获取

#### ❌ 修复前（错误）
```typescript
// main.ts 第119行
Generator.addObject(`mMotorDriver.DCmotor_${motorValue}`, 'Emakefun_DCMotor', `*DCmotor_${motorValue} = mMotorDriver.getMotor(${motorValue});`, true);
```

**生成的C++代码：**
```cpp
// 全局变量区
Emakefun_DCMotor DCmotor_1;  // 声明为对象（非指针）

void setup() {
    *DCmotor_1 = mMotorDriver.getMotor(1);  // ❌ 编译错误：不能解引用非指针
}
```

**问题：** 
- `getMotor()`返回的是指针 `Emakefun_DCMotor*`
- 但变量被声明为对象 `Emakefun_DCMotor`
- 尝试对非指针对象解引用会导致编译错误或运行时崩溃

---

#### ✅ 修复后（正确）
```typescript
// main.ts 第119行
Generator.addObject(`DCmotor_${motorValue}`, 'Emakefun_DCMotor*', `DCmotor_${motorValue} = mMotorDriver.getMotor(${motorValue})`, true);
```

**生成的C++代码：**
```cpp
// 全局变量区
Emakefun_DCMotor* DCmotor_1;  // 声明为指针

void setup() {
    DCmotor_1 = mMotorDriver.getMotor(1);  // ✅ 正确的指针赋值
}
```

**优势：** 
- 类型匹配：指针接收指针
- 符合库的设计意图
- 可以正确使用 `->` 操作符调用方法

### 3. 控制函数中的重复声明

#### ❌ 修复前（错误）
```typescript
// main.ts 第152-158行
export function controlEncoderMotor(parameter: any) {
    let encoderValue = parameter.ENCODER.code;
    let direction = parameter.DIRECTION.code;
    let speed = parameter.SPEED.code;
    
    // ❌ 每次都重新声明对象
    Generator.addObject(`EncodeMotor_${encoderValue}`, 'Emakefun_EncoderMotor', `*EncodeMotor_${encoderValue} = mMotorDriver.getEncoderMotor(${encoderValue});`);
    
    Generator.addCode(`EncodeMotor_${encoderValue}->setSpeed(${speed});`);
    Generator.addCode(`EncodeMotor_${encoderValue}->run(${direction});`);
}
```

**生成的C++代码（如果多次调用）：**
```cpp
void loop() {
    // 第一次调用
    Emakefun_EncoderMotor EncodeMotor_1;
    *EncodeMotor_1 = mMotorDriver.getEncoderMotor(1);
    EncodeMotor_1->setSpeed(200);
    EncodeMotor_1->run(FORWARD);
    
    delay(1000);
    
    // 第二次调用 - ❌ 重复声明！
    Emakefun_EncoderMotor EncodeMotor_1;  // 编译错误：重定义
    *EncodeMotor_1 = mMotorDriver.getEncoderMotor(1);
    EncodeMotor_1->setSpeed(150);
    EncodeMotor_1->run(BACKWARD);
}
```

**问题：** 
- 每次调用都会尝试重新声明变量
- 导致编译错误："redefinition of 'EncodeMotor_1'"
- 即使能编译，也会创建局部变量覆盖全局变量

---

#### ✅ 修复后（正确）
```typescript
// main.ts 第148-158行
export function controlEncoderMotor(parameter: any) {
    let encoderValue = parameter.ENCODER.code;
    let direction = parameter.DIRECTION.code;
    let speed = parameter.SPEED.code;
    
    // ✅ 不再调用addObject，直接使用已声明的全局变量
    Generator.addCode(`EncodeMotor_${encoderValue}->setSpeed(${speed});`);
    Generator.addCode(`EncodeMotor_${encoderValue}->run(${direction});`);
}
```

**生成的C++代码：**
```cpp
// 全局变量区（由initEncoderMotor声明）
Emakefun_EncoderMotor* EncodeMotor_1;

void setup() {
    EncodeMotor_1 = mMotorDriver.getEncoderMotor(1);  // 只声明一次
}

void loop() {
    // 第一次调用
    EncodeMotor_1->setSpeed(200);  // ✅ 直接使用
    EncodeMotor_1->run(FORWARD);
    
    delay(1000);
    
    // 第二次调用 - ✅ 没有重复声明
    EncodeMotor_1->setSpeed(150);
    EncodeMotor_1->run(BACKWARD);
}
```

**优势：** 
- 变量只声明一次（在初始化函数中）
- 控制函数直接使用，无重复声明
- 代码清晰，符合编程规范

### 4. 完整的代码流程对比

#### ❌ 修复前的完整流程

```cpp
#include <Emakefun_MotorDriver.h>

// 全局变量
Emakefun_MotorDriver mMotorDriver;  // 默认构造
Emakefun_DCMotor DCmotor_1;         // 对象（非指针）❌

void setup() {
    Serial.begin(9600);
    mMotorDriver = Emakefun_MotorDriver(0x60);  // 重新赋值 ❌
    mMotorDriver.begin(1000);
    
    // initDCMotor积木生成的代码
    *DCmotor_1 = mMotorDriver.getMotor(1);  // ❌ 解引用错误

    // controlDCMotor积木生成的代码

}
```

// 全局变量

**结果：** 编译成功，电机正常工作！✅

## 总结

| 问题类型 | 修复前 | 修复后 | 影响 |
|---------|--------|--------|------|
| 对象初始化 | `mMotorDriver = Emakefun_MotorDriver(0x60);` | `mMotorDriver(0x60)` | 对象状态可能不正确 |
| 指针声明 | `Emakefun_DCMotor DCmotor_1` | `Emakefun_DCMotor* DCmotor_1` | 类型不匹配，无法使用 |
| 指针赋值 | `*DCmotor_1 = getMotor(1)` | `DCmotor_1 = getMotor(1)` | 解引用错误 |
| 重复声明 | 每次调用都addObject | 只在初始化时addObject | 编译错误：重定义 |

**核心原则：**
1. 对象应该用构造函数初始化，而不是赋值
2. 指针变量必须声明为指针类型（带`*`）
3. 指针赋值不需要解引用操作符（`*`）
4. 全局变量只声明一次，后续直接使用
