## Mind+ 扩展库两种格式对比分析

### 一、文件结构对比

| 维度 | `ext_five_sensor` (自带库格式) | `mindplus_example` (用户库格式) |
|------|-------------------------------|-------------------------------|
| **根配置** | `lib.json` | `config.json` |
| **主入口** | `main.js` | `arduinoC/main.ts` |
| **积木定义** | `emakfun_blocks.js` | `main.ts` 中的 `//% block` 注释 |
| **代码生成** | `emakfun_code.js` | `main.ts` 中的 `Generator` API |
| **工具箱** | `emakfun_toolbox.js` | 由 `main.ts` 自动生成 |
| **多语言** | `language.js` | `_locales/zh-cn.json` |
| **下拉菜单** | 硬编码在 blocks.js 中 | `_menus/主板名.json` |
| **库文件** | 无（需用户手动安装） | `libraries/libraries.zip` |

### 二、核心差异详解

#### 1. 自带库 (`ext_five_sensor`) - 旧版格式

```javascript
// lib.json - 扩展库配置
{
    "id": "ext_five_infrared_tracking",
    "blocks": "arduino_uno_blocks.js",      // 积木外观定义
    "blocks_code": "arduino_uno_code.js",   // 代码生成器
    "blocks_xml": "arduino_xml",            // 工具箱XML
    "hardType": "atmega328p",               // 单一主板支持
    "category": "Arduino"
}
```

**积木定义方式**（emakfun_blocks.js）：
```javascript
Blockly.Blocks['EXT_FIVE_INFRARED_TRACKING_INIT'] = {
    init: function () {
        this.jsonInit({
            "message0": "五路巡线模块初始化IIC接口",
            "colour": '#28BFE6',
            "extensions": ["shape_statement"]
        });
    }
};
```

**代码生成方式**（emakfun_code.js）：
```javascript
Blockly.Arduino['EXT_FIVE_INFRARED_TRACKING_INIT'] = function (block) {
    Blockly.Arduino.definitions_['include'] = `#include "InfraredTracking.h"`;
    return `_5line_track.Init();\n`;
}
```

#### 2. 用户库 (`mindplus_example`) - 新版格式

```json
// config.json - 扩展库配置
{
    "id": "EM_five_tracker_V3",
    "board": ["arduino", "esp32", "mega2560"],  // 多主板支持
    "asset": {
        "arduinoC": {
            "main": "main.ts",
            "files": ["_locales/", "_menus/", "libraries/"]
        }
    }
}
```

**积木定义与代码生成合一**（main.ts）：
```typescript
//% block="five tracker v3 set [INDEX] high threshold [LINENUMBER]" blockType="command"
//% INDEX.shadow="dropdown" INDEX.options="INDEXPROBE" INDEX.defl="INDEXPROBE.0"
export function setSensorHighThresholdValue(parameter: any, block: any) {
    Generator.addInclude(`fiveTrackerInitV3`, `#include <five_line_tracker_v3.h>`);
    Generator.addSetup(`line_tracker_setup_v3`, `Wire.begin();\n  line_tracker_v3.Initialize();`);
    Generator.addCode(`line_tracker_v3.HighThreshold(${index}, ${lieNumber});`);
}
```

```
┌─────────────────────────────────────────────────────────────┐
│                    Mind+ 扩展库体系                          │
├───────────────────────────┬─────────────────────────────────┤

    Mind+ 内置加载                    本地导入/网络加载
```

### 四、能否相互转化？

**结论：可以单向转化（自带库 → 用户库），但反向转化有限制。**

#### 1. 自带库 → 用户库（可行，需手动迁移）

| 迁移项 | 自带库 → 用户库 | 说明 |
|--------|----------------|------|
| 积木定义 | blocks.js → main.ts `//% block` | 需将 JSON 格式转为 TS 注解 |
| 代码生成 | code.js → main.ts `Generator` | 直接迁移逻辑，改写成 TS |
| 多语言 | language.js → _locales/zh-cn.json | 格式转换 |
| 下拉菜单 | 硬编码 → _menus/主板.json | 提取为独立 JSON |
| 库依赖 | 无 → libraries/libraries.zip | 需手动打包 C++ 库 |

**迁移示例**：

自带库 blocks.js：
```javascript
Blockly.Blocks['EXT_FIVE_INFRARED_TRACKING_INIT'] = {
    init: function () {
        this.jsonInit({
            "message0": "五路巡线模块初始化IIC接口",
            "colour": '#28BFE6',
            "extensions": ["shape_statement"]
        });
    }
};
```

转化为用户库 main.ts：
```typescript
//% block="五路巡线模块初始化IIC接口" blockType="command" color="#28BFE6"
export function initSensor(parameter: any, block: any) {
    Generator.addInclude(`init`, `#include "InfraredTracking.h"`);
    Generator.addCode(`_5line_track.Init();`);
}
```

#### 2. 用户库 → 自带库（不推荐，功能损失）

**限制原因**：
- 用户库的 `Generator.board` 多主板判断无法映射到自带库
- `_menus/` 下拉菜单配置无法还原到 JS 硬编码
- `libraries.zip` 无法自动释放到用户 Arduino 库目录

---

### 五、建议迁移方案

如果你的 `ext_five_sensor` 需要转为用户库格式：

1. **创建新目录结构**
   ```
   new_five_sensor/
   ├── config.json
   └── arduinoC/
       ├── main.ts
       ├── _locales/zh-cn.json
       ├── _menus/arduino.json
       └── libraries/libraries.zip
   ```

2. **重写 main.ts**
   - 将 `emakfun_blocks.js` 中的 `jsonInit` 转为 `//% block` 注解
   - 将 `emakfun_code.js` 中的 `Blockly.Arduino['xxx']` 转为 `Generator.addXxx()` 调用

3. **分离配置**
   - 中文文本 → `_locales/zh-cn.json`
   - 下拉选项 → `_menus/arduino.json`
   - C++ 库文件 → `libraries/libraries.zip`

4. **本地测试**
   - Mind+ → 扩展 → 用户库 → 导入本地库 → 选择 `config.json`

### 六、总结

| 问题 | 答案 |
|------|------|
| 两种库什么关系？ | 自带库是旧版格式，用户库是新版(V1.6.2+)推荐格式 |
| 能否互相转化？ | 自带库→用户库 ✅ 可行；用户库→自带库 ❌ 不推荐 |
| 官方推荐用哪种？ | 用户库（.mpext 格式），支持导出分享和多主板 |
| 迁移工作量？ | 中等，主要是格式转换和文件重组 |

**建议**：如果是新项目，直接使用用户库格式；如果是维护旧项目，可考虑迁移以获得更好的多主板支持和分享功能。
